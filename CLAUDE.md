# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a literate Emacs configuration using org-babel. `init.el` is minimal — it calls `org-babel-load-file` on `readme.org`, which contains all actual configuration as `emacs-lisp` code blocks. **All config changes go in `readme.org`**, not `init.el`.

**Important:** Do not rename `readme.org` to `init.org` — tangling an org file named `init.org` would overwrite `init.el`.

## Architecture

The entire configuration lives in `readme.org` as a single org file with sections:

- **Package management** — MELPA + `use-package` with `:ensure t` by default
- **Generic configuration** — `is-mac` / `is-linux` variables used for OS branching; `custom.el` kept separate; backup/auto-save redirected to `auto-backup/` and `auto-save/`
- **Appearance** — powerline modeline, catppuccin theme (light) or material (dark) controlled by `light-theme`/`dark-theme` variables at top of Generic configuration section; font is JetBrainsMono Nerd Font on both platforms
- **Major modes** — org/org-roam/org-journal, nix-mode, python, ruby/enh-ruby-mode, magit, yaml, go-mode, rust-mode, ediff, asciidoc
- **Minor modes** — which-key, lsp-mode + lsp-ui (hooked to go-mode), company, yasnippet, ivy/swiper/counsel, markdown, flycheck (+yamllint/bashate/rust), ansible, jinja2
- **Other packages** — ripgrep (`rg`), wgrep
- **Custom functions** — `buffer-list-moveto` bound to `C-x C-b`

## Notes

- `custom.el` is git-ignored; it is auto-generated by Emacs and loaded at startup
- `elpa/` is git-ignored; packages are auto-installed on first startup via `use-package`
- `readme.el` is git-ignored; it is the tangled output Emacs caches from `readme.org`
- org-roam notes live at `~/Documents/Notes/Roam`, org-journal at `~/Documents/Notes/Journal`
- SSH agent socket is imported from shell environment via `exec-path-from-shell` on macOS
